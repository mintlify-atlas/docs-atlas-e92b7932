Functions can be called to generate content. Functions are called by their name followed by parentheses (`()`) and may have arguments.

For instance, the `range` function returns a list containing an arithmetic progression of integers:

```canvas
{% for i in range(0, 3) %}
  {{ i }},
{% endfor %}
```

## Available functions

Canvas provides a variety of functions for different purposes:

### Template functions

- [include](/canvas/functions/include) - Include and render a template
- [source](/canvas/functions/source) - Return template source code
- [block](/canvas/functions/block) - Render a block
- [parent](/canvas/functions/parent) - Render parent block content

### Utility functions

- [attribute](/canvas/functions/overview) - Access dynamic attributes
- [constant](/canvas/functions/constant) - Access constant values
- [cycle](/canvas/functions/cycle) - Cycle through values
- [date](/canvas/functions/date) - Create date objects
- [dump](/canvas/functions/dump) - Debug variable output
- [max](/canvas/functions/max) - Get maximum value
- [min](/canvas/functions/min) - Get minimum value
- [random](/canvas/functions/random) - Generate random values
- [range](/canvas/functions/range) - Generate number sequences

### Blutui-specific functions

- [asset](/canvas/functions/asset) - Generate asset URLs
- [agent](/canvas/functions/agent) - Get user agent information
- [recaptcha](/canvas/functions/recaptcha) - Generate reCAPTCHA widget

### Canopy functions

Canopy functions provide editable components for the Canopy visual editor:

- [cms_text](/canvas/functions/canopy/cms-text) - Editable text component
- [cms_heading](/canvas/functions/canopy/cms-heading) - Editable heading component
- [cms_image](/canvas/functions/canopy/cms-image) - Editable image component
- [cms_button](/canvas/functions/canopy/cms-button) - Editable button component
- [cms_code](/canvas/functions/canopy/cms-code) - Editable code component
- [cms_list](/canvas/functions/canopy/cms-list) - Editable list component

## Named arguments

```canvas
{% for i in range(low = 1, high = 10, step = 2) %}
  {{ i }},
{% endfor %}
```

Using named arguments makes your templates more explicit about the meaning of the values you pass as arguments:

```canvas
{{ data | convert_encoding('UTF-8', 'iso-2022-jp') }}

{# versus #}

{{ data | convert_encoding(from = 'iso-2022-jp', to = 'UTF-8') }}
```

Named arguments also allow you to skip some arguments for which you don't want to change the default value:

```canvas
{# the first argument is the date format, which defaults to the global date format if null is passed #}
{{ 'now' | date(null, 'Europe/Paris') }}

{# or skip the format value by using a named argument for the time zone #}
{{ 'now' | date(timezone = 'Europe/Paris') }}
```

You can also use both positional and named arguments in one call, in which case positional arguments must always come before named arguments:

```canvas
{{ 'now' | date('d/m/Y H:i', timezone = 'Europe/Paris') }}
```
