---

This guide will walk you through creating your first Blutui project, setting up local development with Courier, and deploying your changes.

## Prerequisites

Before you begin, make sure you have:

- A [Blutui account](https://auth.blutui.com/register)
- Node.js and npm installed (for most starter templates)
- Basic familiarity with HTML and terminal commands

<Steps>

<Step title="Create a Blutui Account">

If you don't already have one, [sign up for a Blutui account](https://auth.blutui.com/register). Development projects are free and password-protected.

</Step>

<Step title="Create Your First Project">

1. Login to your [Agency Console](https://console.blutui.com)
2. Click **Projects** in the navigation menu
3. Click **Create project**
4. Enter a **Project name** (e.g., "my-first-project")
5. Optionally add a description and set your timezone
6. Click **Create project**

<Info>
Development projects are free and password-protected. Production projects require a monthly subscription.
</Info>

</Step>

<Step title="Install Courier CLI">

Courier is Blutui's command-line tool for local development. Choose your operating system:

<CodeGroup>

```bash macOS
brew install --cask blutui/courier/courier
```

```bash Windows
scoop bucket add blutui https://github.com/blutui/bucket-courier.git
scoop install blutui/courier
```

```bash Linux (Debian/Ubuntu)
# Download .deb from https://blutui.com/developers/courier
sudo apt install ./courier_*_linux_amd64.deb
```

</CodeGroup>

<Note>
For detailed installation instructions, see the [Courier installation guides](/courier/installation/macos).
</Note>

</Step>

<Step title="Create a Courier Token">

1. In your [Agency Console](https://console.blutui.com), click your avatar in the top-right
2. Choose **Edit my profile**
3. Go to the **Apps** section in the sidebar
4. Click **Create token** under "Access tokens"
5. Name your token (e.g., "Work MacBook")
6. Set the token type to **Courier**
7. Click **Create token** and copy the token (it's only shown once)

</Step>

<Step title="Login to Courier">

Create a temporary file with your token and login:

```bash
echo "your-token-here" > token.txt
courier login --token < token.txt
rm token.txt
```

</Step>

<Step title="Pull Your Project">

Create a directory for your project and initialize it:

```bash
mkdir my-first-project
cd my-first-project
courier init my-first-project
courier pull
```

This creates a `courier.json` configuration file and pulls your project files into `public/` and `views/` directories.

</Step>

<Step title="Start Local Development">

If your project uses a build tool (like the tailwindcss-js template), install dependencies first:

```bash
npm install
npm run dev
```

Then in a separate terminal, start Courier's development server:

```bash
courier dev
```

Your project will be available at `https://localhost:8080/`

<Info>
Courier dev watches for file changes in `public/` and `views/` and automatically syncs them to Blutui.
</Info>

</Step>

<Step title="Make Your First Change">

Edit a file in the `views/` directory. For example, create or edit `views/layouts/default.canvas`:

```canvas views/layouts/default.canvas
{% block body %}
  <section class="bg-white text-slate-900 p-8">
    {{ cms_heading('hero-heading', {
      element: 'h1',
      value: 'Welcome to My First Blutui Project',
      class: 'text-4xl font-bold'
    }) }}
    
    {{ cms_text('hero-description', {
      value: 'This is my first project built with Blutui!',
      class: 'text-lg mt-4'
    }) }}
  </section>
{% endblock %}
```

Courier will automatically sync your changes.

</Step>

<Step title="Push Your Changes">

When you're ready to deploy your changes to the staging environment:

```bash
courier push
```

Your project will be available at `https://your-project-handle.blutui.com`

</Step>

</Steps>
