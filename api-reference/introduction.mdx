## Overview

Blutui provides a comprehensive REST API to programmatically manage your projects, content, and infrastructure. The API is organized into three main categories:

- **Admin API** - Manage project content including blogs, posts, collections, forms, and more
- **Console API** - Manage agency resources, projects, domains, and team members
- **User API** - Access authenticated user information
- **Agencies API** - Manage agency settings and information

## Base URLs

All API requests use HTTPS. HTTP requests are not supported.

<CodeGroup>
```text Admin API
https://{handle}.blutui.com/admin/api
```

```text Console API
https://api.blutui.com/v1
```
</CodeGroup>

Replace `{handle}` with your project's unique handle for Admin API requests.

## SDKs

Blutui provides official SDKs to make integration easier:

### Node.js SDK

Install the official Node.js SDK:

```bash
npm install blutui
```

Initialize the SDK:

```javascript
import Blutui from 'blutui'

const blutui = new Blutui('your-access-token')

// Admin API
await blutui.project('handle').admin.blogs.list()

// Console API
await blutui.agency('username').projects.list()

// User API
await blutui.user.get()
```

## Response Format

All API responses use JSON format. Successful responses return a `200` status code along with the requested data. List endpoints return paginated results with metadata.

## Error Handling

The API uses standard HTTP status codes:

- `200` - Success
- `204` - No Content (resource doesn't exist)
- `401` - Unauthorized (invalid or missing access token)
- `403` - Forbidden (insufficient permissions)
- `404` - Not Found
- `422` - Validation Error

## Rate Limiting

API requests are subject to rate limiting to ensure platform stability. Rate limit information is included in response headers.

## Getting Started

1. [Generate an access token](https://console.blutui.com/profile) from your Blutui account
2. Review the authentication documentation
3. Choose the appropriate API for your use case
4. Make your first API request
