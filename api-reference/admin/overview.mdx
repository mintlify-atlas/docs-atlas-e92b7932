## Overview

The Blutui Admin API is built on **REST** principles. We enforce **HTTPS** in every request to improve data security and privacy. The API does not support **HTTP**.

All requests contain the following base URL, where `handle` is your project's handle:

```text Admin API Base URL
https://{handle}.blutui.com/admin/api
```

## Authentication

The Admin API requires a Blutui [access token](https://console.blutui.com/profile) to authenticate requests. Include your token in the `Authorization` header:

```bash
curl https://{handle}.blutui.com/admin/api/blogs \
  -H 'Authorization: Bearer ey....'
```

Learn more in the [Authentication documentation](/api-reference/authentication).

## Using the Node.js SDK

The official Node.js SDK provides a clean interface for the Admin API:

```javascript
import Blutui from 'blutui'

const blutui = new Blutui('ey....')

// List all blogs
const blogs = await blutui.project('one').admin.blogs.list()

// Create a new blog post
const post = await blutui.project('one').admin.posts.create({
  title: 'Hello World',
  slug: 'hello-world',
  content: '<p>Welcome to my blog!</p>',
  author: 'John Doe',
  blog: 'blog-id',
  publish_date: '2024/09/25 11:00:00'
})

// Update a collection entry
const entry = await blutui.project('one').admin.entries.update(
  'collection-id',
  'entry-id',
  { title: 'Updated Title' }
)
```

## Pagination

List endpoints support pagination with the following query parameters:

<ParamField query="limit" type="integer" default={10}>
  Limit the number of objects to return, ranging between 1 and 20.
</ParamField>

<ParamField query="page" type="integer" default={1}>
  The current page used to display the pagination results.
</ParamField>

## Expanding Resources

Many endpoints support expanding related resources using the `expand` query parameter:

```bash
curl 'https://{handle}.blutui.com/admin/api/blogs?expand[]=list' \
  -H 'Authorization: Bearer ey....'
```

Available expand options vary by endpoint. Check individual endpoint documentation for details.
